# Release

## Procedure
1. **Pre-release** - Clean working tree, all tests pass
2. **Version** - Review changes since last tag, apply semver (MAJOR/MINOR/PATCH)
3. **Changelog** - Group by type (Added, Changed, Fixed, Removed)
4. **Release** - Update version files, create git tag `vX.Y.Z`
5. **Summary** - Version bump, key changes, next steps
